# PRD — v0.5.0 (CSS 2026 theme)

## Summary
v0.5.0 focuses on shipping modern, low-JS UI components that leverage stable and near-stable CSS features from the 2026 landscape. Emphasis is on progressive enhancement, SSR-friendly HTML, and accessibility-first defaults.

## Strategic goals
- Deliver a production-ready Carousel component (carousel-pk) using CSS Scroll Snap and keyboard navigation.
- Ship a date-picker (date-picker-pk) that enhances native inputs and falls back gracefully to a fully accessible calendar popover.
- Improve text layout by adopting `text-wrap: balance` where appropriate and documenting graceful degradation.
- Run a targeted CSS `if()` refactor pass to centralize environment-specific fallbacks.

## Non-goals
- Do not add heavy JS libraries. Keep runtime minimal and framework-agnostic.
- Do not change component naming conventions or file layout.

## MVP Scope
- carousel-pk (Must):
  - Requirements: SSR markup that renders sensibly without JS, keyboard focus ring, pagination dots accessible, touch + scroll snap support.
  - Accessibility: announce slide changes using aria-live region; prefer `role="group"` with accessible labeling.
  - Tests: unit (vitest) + e2e (Playwright) across Chromium/Firefox/WebKit.

- date-picker-pk (Should):
  - Requirements: enhance <input type="date">, provide calendar popover for browsers without good native support, ensure keyboard nav and screen reader announcement.
  - Fallback: use accessible listbox for date selection when popover/anchor API unavailable.

- color-swatch-pk (Could):
  - Requirements: present selectable swatches, keyboard focus, aria-selected, and contrast guidelines.

## QA & Success Criteria
- Unit coverage: 100% for new client modules.
- Playwright e2e: All scenarios pass on Chromium, Firefox, and WebKit.
- Accessibility: axe-core audits show no critical or serious violations for component demos.

## Implementation notes
- Use existing patterns from pseudo-assets components (slot-based options, minimal JS in <script> block).
- Follow migration patterns from S11/S13 for Listbox/Combobox accessibility (aria-activedescendant, option ids).
