# Story S9-04 — Notification: autodismiss animation + Interest Invokers

**Epic:** Migration
**Sprint:** Sprint 09
**Points:** 3
**Priority:** Must
**Status:** ✅ DONE

## User Story

As a **vanilla JS developer**, I want notifications to auto-dismiss after a configurable duration using CSS animation, and pause on hover/focus, so that I don't need JavaScript timers for this behavior.

## Context

Current notification is static — it renders and stays until manually removed. This story adds an `[autodismiss]` attribute that triggers a CSS fade-out animation. Hover/focus pauses the animation via `:hover`/`:focus-within` (all browsers) and `:interest` (Chrome 146+). See tech-spec section 4 (Notification).

Traces: MIG-09, MIG-10

## Implementation Summary

### Template Changes
- No changes — notification structure remains the same

### CSS Changes
- Added `@keyframes pk-notif-dismiss` animation (opacity 1→0, slideUp)
- Applied animation only when `[autodismiss]` attribute is present
- Pause animation on `:hover` and `:focus-within` (all browsers)
- Added `@supports selector(:interest)` block for Chrome 146+ enhancement
- CSS custom property `--dismiss-duration` controls animation length (default 5s)

### Script Changes
- Listen to `animationend` event on `.notification` element
- Remove element from DOM when animation completes
- Handle dismiss button clicks to remove element immediately
- Clean up animation listener if `[autodismiss]` attribute is removed during animation

### Test Results
- All 51 client tests pass ✅
- No regressions — notification maintains existing test coverage
- Autodismiss feature is additive; existing static notifications unchanged
