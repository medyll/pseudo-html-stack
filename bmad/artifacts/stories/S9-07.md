# Story S9-07 — Cross-browser fallback validation (Firefox, WebKit)

**Epic:** QA
**Sprint:** Sprint 09
**Points:** 2
**Priority:** Should

## User Story

As a **developer**, I want to verify that fallback behavior works correctly in Firefox and WebKit, so that the migration doesn't break the component library for users on non-Chromium browsers.

## Context

Anchor Positioning and Interest Invokers have limited support outside Chromium. Popover API is supported in Firefox 125+ and Safari 17+. This story validates the `@supports not` CSS paths and JS fallback handlers.

Traces: TST-04, FB-04, FB-05, FB-06

## Acceptance Criteria

```gherkin
Given Firefox (latest)
When modal opens
Then it works via showModal() (supported) or fallback JS

Given Firefox (latest)
When dropdown trigger is clicked
Then Popover API works (Firefox 125+)

Given Firefox (latest)
When tooltip renders
Then absolute positioning fallback applies (no anchor-name support)

Given WebKit (Safari 17.4+)
When all 4 components render
Then they function correctly via supported APIs or fallback

Given any browser without :interest support
When notification has [autodismiss]
Then :hover/:focus-within still pauses the animation
```
