**Epic:** Components

**Points:** 3
**Priority:** Could
**Status:** done

## User Story
As a developer, I want a `progress-pk` atom that wraps the native `<progress>` element with variant, size, and indeterminate support, so that I can show accessible progress indicators without custom JS or fake divs.

## Acceptance Criteria

```gherkin
Given a progress-pk with value and max
When the page loads
Then the inner progress element has value and max set

Given a progress-pk with no value attribute
When the page loads
Then the component is in indeterminate state (no value on <progress>, indeterminate attr on host)

Given a progress-pk with variant and size props
When inspecting the host element
Then the variant and size attributes are present for CSS @scope targeting

Given a progress-pk
When component initializes
Then data-ready="true" is set on the host
```

## Technical Notes
- Wraps native `<progress>` element (accessibility built-in)
- Props: `value` (omit for indeterminate), `max` (default 100), `variant`, `size`, `label`
- CSS: `@scope (.progress)` with `::-webkit-progress-value` / `::-moz-progress-bar` vendor pseudo-elements
- Indeterminate animation via `@keyframes` when `[indeterminate]` attribute present on host
