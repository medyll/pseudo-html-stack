# Story S10-07 — E2E + Unit Tests (Form & Layout Components)

**Epic:** Testing & QA  
**Sprint:** Sprint 10  
**Points:** 4  
**Priority:** Must  

---

## User Story

As a **qa engineer**, I want comprehensive unit and E2E test coverage for form and layout component migrations (S10-01, S10-03, S10-05), ensuring fallback behavior works across all browsers.

## Context

S10-01, S10-03, and S10-05 introduce new CSS APIs (constraint validation, Container Queries, Grid Lanes). This story creates test suite with browser-conditional paths: native API paths in Chrome 146+, fallback paths in older browsers.

```gherkin
When running migrations-e2e-forms.e2e.js

  - Input error span displays

```

## Technical Notes

- Unit tests: vitest + happy-dom + mocks for Popover, Container Queries
- E2E tests: Playwright, 3 browsers (Chromium, Firefox, WebKit)
- Browser-conditional: `test.skip(browserName === 'chromium', ...)` for fallback-only assertions
- Mock APIs: Add to `vitest.config.js` or test setup
- Coverage target: 100% on `src/client/**` (maintain baseline)

## Out of Scope

- Pixel-perfect visual regression
- Legacy browser testing (IE11, Chrome < 118)
- Performance benchmarks (speed tests deferred to post-release)

## Implementation Plan

**Phase 1: Unit Tests (vitest)**
1. Create test files for S10-01, S10-03, S10-05
2. Add mocks for Popover, Container Queries
3. Write assertions for both native + fallback paths
4. Run `npm run test:client` — aim for 65+ tests

**Phase 2: E2E Tests (Playwright)**
1. Create `tests/migration-e2e-forms.e2e.js`
2. Add input validation, checkbox, grid test suites
3. Browser-conditional paths: `test.skip(browserName === 'chromium', ...)`
4. Run `npx playwright test` — aim for 12+ E2E tests
5. Validate cross-browser: all 3 projects passing

**Phase 3: Final QA**
1. Run full test suite: `npm run test:all`
2. Verify coverage: `npm run test:coverage` (must be 100%)
3. Check console: no errors or warnings
4. Accessibility: `npm run test:a11y` (axe-core)
