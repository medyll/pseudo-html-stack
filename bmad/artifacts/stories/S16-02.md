**Points:** 3
**Priority:** Should
**Status:** done

## User Story
As a developer, I want to open the canvas viewer with `?canvas=my-component.html&watch=1` and have it automatically reload when the file changes, so that I can iterate on components with live visual feedback.

## Implementation Notes

**Date:** 2026-03-11
**Files changed:**
- `src/pseudo-canvas/viewer/pseudo-canvas-viewer.html` — extended `?canvas=` block with `?watch=1` polling

**Approach:**
- `?watch=1` triggers a `setTimeout`-based poll (1s interval) using `fetch HEAD` with `cache: no-store`
- Compares `ETag` and `Last-Modified` response headers; reloads on change
- Shows `👁 Watching for changes…` toast on start
- Shows `🔄 Reloaded` toast on each reload
- Falls back silently when server doesn't send cache headers

**Known limitations:**
- Only works over http:// (not file://) since fetch HEAD requires a server
- No WebSocket support — polling at 1s is sufficient for local dev
