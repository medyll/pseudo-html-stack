# Story S10-03 — Checkbox/Radio: :checked/:invalid States + Indeterminate

**Epic:** Form Controls Migration  
**Sprint:** Sprint 10  
**Points:** 3  
**Priority:** Must  

---

## User Story

As a **form developer**, I want checkboxes and radio buttons to update visual state via CSS `:checked` and `:invalid` pseudo-classes, eliminating the need for JS attribute watchers.

## Context

Current `checkbox-pk.html` and `radio-pk.html` use inline scripts to toggle visual classes. Native `:checked` pseudo-class handles this via CSS only. Additional: `HTMLInputElement.indeterminate` property for tri-state checkboxes.

## Acceptance Criteria

```gherkin
Given checkbox-pk [type="checkbox"]
When user clicks checkbox
Then :checked pseudo-class applies automatically
  And label visual state updates (no JS)
  
Given radio-pk [type="radio"] with [name="group"]
When user selects one radio
Then :checked applies to selected radio
  And :not(:checked) applies to siblings
  And form.elements[name].value === selected value
  
Given checkbox with [required] + unchecked
When form submitted
Then :invalid pseudo-class applies
  And error styling visible
  
Given tri-state checkbox (indeterminate)
When element.indeterminate = true
Then visual state reflects third state (-)
  And form submission treats as unchecked
```

## Technical Notes

- `:checked` pseudo-class: no JS needed for visual updates
- `:invalid` for required checkboxes + radio groups
- `HTMLInputElement.indeterminate` property (read/write) for tri-state
- `aria-checked` for accessibility (alternative for :checked detection)
- Fallback: JS watches `change` event to update class (older browser path)

## Out of Scope

- Custom checkbox/radio redesigns (design-only task)
- Toggle switches (different component)
- Multi-select checkboxes with shared state

## Dependencies

- None (independent)

## Definition of Done

- [x] `checkbox-pk.html` updated: :checked CSS only
- [x] `radio-pk.html` updated: :checked + radio group behavior
- [x] Indeterminate state working (tri-state checkboxes)
- [x] Unit tests passing (mock :checked in vitest)
- [x] E2E tests cover checked/unchecked/indeterminate states
- [x] No console errors
- [x] A11y: `aria-checked` + role maintained

## Implementation Notes

**Removed:** Inline script that applied `.checked` class on input change  
**Added:**
- CSS: `input:checked { ... }` and `input:not(:checked) { ... }` selectors
- CSS: `input:invalid` for validation styling
- JavaScript (minimal): Only `element.indeterminate = true` setter for tri-state
- No change listeners needed for visual state

**Browsers:**
- All modern browsers support `:checked`, `:invalid` (CSS 2.1+)
- `indeterminate` property supported in all modern browsers
