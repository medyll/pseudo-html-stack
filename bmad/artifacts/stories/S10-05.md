# Story S10-05 — Grid: CSS Grid Lanes + Container Queries (Masonry Layout)

**Epic:** Layout Components Migration  
**Sprint:** Sprint 10  
**Points:** 5  
**Priority:** Must  

---

## User Story

As a **layout developer**, I want to build masonry layouts using CSS Grid Lanes and Container Queries, eliminating JS layout calculations and media-query breakpoints.

## Context

Grid Lanes (CSS 2026 spec) enable multi-column masonry without JavaScript. Container Queries allow responsive behavior based on container width, not viewport. Current `grid-pk.html` uses flexbox + JS. This story adds Grid Lanes with `@supports` fallback to flexbox.

## Acceptance Criteria

```gherkin
Given grid-pk with [display="lanes"]
When rendered in modern browser (Chrome 146+)
Then CSS Grid Lanes applies
  And items flow into equal-height columns
  And no JS layout calculation runs
  
Given grid-pk container [container-type="inline-size"]
When container width changes
Then @container (min-width: ...) { } queries apply
  And child items reflow without media query
  
Given browser without Grid Lanes support
When grid-pk rendered
Then @supports (display: grid-lanes) fallback activates
  And flexbox grid displays with equal-height items
  
Given grid-pk with gap and padding props
When container queries change size
Then CSS custom properties (--gap, --cols) update
  And layout recalculates smoothly (no reflow jank)
```

## Technical Notes

- `display: grid-lanes` (2026 spec, Chrome 146+)
- `container-type: inline-size` for size-based queries
- `@container` blocks for responsive behavior
- `@supports (display: grid-lanes)` for graceful degradation
- Fallback: `display: flex` + `flex-wrap: wrap` with `align-content: start`
- Custom properties: `--cols`, `--gap`, `--item-width` for responsive tuning

## Out of Scope

- Grid template areas (grid-template-areas syntax)
- Subgrid (nested grid containers)
- Masonry plugin polyfills for older browsers

## Implementation Notes

**New component:** `src/pseudo-assets/components/molecules/grid-pk.html`

**Template structure:**
```html
<div grid-pk [display="lanes"] [container-type="inline-size"]>
  <div class="grid-item">Item 1</div>
  <div class="grid-item">Item 2</div>
  ...
</div>
```

**CSS:**
```css
@supports (display: grid-lanes) {
  @scope (grid-pk) {
    :scope { display: grid-lanes; /* 2026 spec */ }
  }
}

@supports not (display: grid-lanes) {
  /* Fallback to flexbox */
  @scope (grid-pk) {
    :scope { display: flex; flex-wrap: wrap; }
  }
}

@supports (container-type: inline-size) {
  grid-pk { container-type: inline-size; }
  @container (min-width: 768px) {
    /* responsive adjustments */
  }
}
```

**Browser support:**
- Chrome 146+: Grid Lanes + Container Queries (full native)
- Firefox 128+, Safari 17.4+: Flexbox fallback (no Grid Lanes yet)
