# S13-03 Fix Plan — Migration E2E Flaky Tests (Chromium)

Scope
-----
Implement deterministic fixes for remaining migration-e2e Chromium failures: dropdown/menu visibility, notification autodismiss/dismiss behavior, and modal/dropdown interaction overlay.

Planned changes (short-term, non-invasive first):
1. Dropdown
   - Ensure dropdown trigger exists and is interactive before click by adding data-ready or waiting for computed style/popover state in tests.
   - If component is at fault, add `data-ready` on dropdown root when popover target is mounted.
2. Notification
   - Expose `data-ready` or `data-interactive` when dismiss button is visible; ensure autodismiss timer clears on hover.
   - In tests, wait for dismiss button presence and use force-click fallback only when necessary.
3. Integration overlay
   - Prefer opening dropdown before modal in tests, or ensure modal has `pointer-events: none` on backdrop content when not intended to intercept.
   - If component overlay intercepts unexpectedly, adjust z-index or add `[data-overlay-active]` hook to signal blocking.

Tasks
-----
- Implement `data-ready` hooks in components (if safe) or add deterministic waits in tests.
- Update migration-e2e.e2e.js to use readiness checks (no broad timeouts).
- Run local headed Chromium tests to confirm (developer to run manually per policy).
- Prepare PR with changes and include test run artifacts in PR description.
