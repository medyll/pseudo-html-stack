# S14-03 — tabs-pk organism — ARIA Tabs + View Transitions

**Epic:** Components
**Sprint:** Sprint 14
**Points:** 5
**Priority:** Should
**Status:** done

## User Story
As a developer, I want a `tabs-pk` organism that wires tab navigation to tab panels with full ARIA support and keyboard navigation, so that I can build accessible tabbed interfaces without writing custom JS.

## Acceptance Criteria

```gherkin
Given a tabs-pk with tab buttons and panels
When the page loads
Then the first tab is active and its panel is visible

Given a tabs-pk
When the user clicks a tab button
Then the corresponding panel becomes visible and others hide

Given a tabs-pk
When the user presses ArrowRight on an active tab
Then focus moves to the next tab

Given a tabs-pk with data-testid
When component initializes
Then data-ready="true" is set on the host
```

## Technical Notes
- Host: `<tabs-pk>` organism
- Slot `tabs` — list of `<button>` tab triggers; each needs `data-panel="id"` to link to a panel
- Slot `panels` — list of `<div role="tabpanel">` elements with matching `id`
- Script wires ARIA (`aria-selected`, `aria-controls`, `aria-labelledby`, `tabindex`)
- View Transitions API (`document.startViewTransition`) for smooth panel switch if available
- Keyboard: `ArrowRight`, `ArrowLeft`, `Home`, `End`
- Falls back gracefully when View Transitions not available

## Out of Scope
- Vertical tabs orientation (future story)
- Lazy panel loading
