# Story S10-01 — Input: HTML5 Constraint Validation + Popover Validation Hints

**Epic:** Form Controls Migration  
**Sprint:** Sprint 10  
**Points:** 5  
**Priority:** Must  

---

## User Story

As a **form developer**, I want to use HTML5 constraint validation API instead of custom JS validators, so that the browser handles validation natively with zero additional script.

## Context

HTML5 provides native validation via `required`, `pattern`, `min`, `max`, and constraint validation API. Current `input-pk.html` uses custom JS to validate. This story migrates to native APIs with Popover hints for error messaging.

## Acceptance Criteria

```gherkin
Given input-pk with [required] attribute
When form is submitted without value
Then :invalid pseudo-class applies (no JS)
  And browser-native error message shows
  
Given input-pk with [pattern] or [min]/[max]
When value violates constraint
Then input.validity.valid === false
  And error popover shows via popovertarget
  
Given input-pk with validation error
When user corrects value
Then input:valid applies
  And error popover auto-dismisses
  
Given older browser without Popover API
When validation fails
Then fallback error <span> shows (no JS error)
```

## Technical Notes

- Use `HTMLFormElement.checkValidity()` and `input.validity` object
- Error messages via Popover API (`popover="auto"` with `popovertarget`)
- Fallback: `display:none` error span + JS event listeners (Safari < 17 path)
- CSS: `:invalid` + `:valid` + `:user-invalid` for visual feedback
- Test with happy-dom mock (Popover API not in happy-dom)

## Out of Scope

- Custom validator plugins
- Async validation (email verification, username availability)
- Form-level validation (multi-field dependencies)

## Dependencies

- None (parallel execution safe)

## Definition of Done

- [x] `input-pk.html` updated with constraint validation
- [x] Popover API integration for error hints
- [x] Fallback :invalid CSS + error span for older browsers
- [x] Unit tests passing (mock Popover API in vitest)
- [x] E2E tests cover native + fallback paths
- [x] No console errors or warnings
- [x] A11y: error messages linked to input via `aria-describedby` + popover `role="alert"`

## Implementation Notes

**Deleted inline script:** Custom validation logic removed from component  
**Added:**
- `<input required pattern="..." min="..." max="...">`
- Constraint validation API: `element.validity`, `element.checkValidity()`
- Popover hint: `<div popover="auto" id="error-{id}">` + `popovertarget="{id}"`
- CSS `:invalid` + `:user-invalid` for styling
- Fallback: `<span class="error" hidden>` displayed via JS when Popover unavailable

**Browser support:**
- Chrome 118+, Firefox 128+, Safari 17.4+: full native
- Older browsers: fallback span + JS listeners
