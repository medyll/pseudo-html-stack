# S15-01 — slider-pk atom — native `<input type="range">` + custom CSS

**Epic:** Components
**Sprint:** Sprint 15
**Points:** 3
**Priority:** Must
**Status:** done

## User Story
As a developer, I want a `slider-pk` atom that wraps the native range input with accessible labels, min/max/step support, and consistent cross-browser styling, so that I can build value-selection controls without custom JS or browser inconsistencies.

## Acceptance Criteria

```gherkin
Given a slider-pk with min, max, step and value props
When the page loads
Then the inner input[type="range"] has those attributes set correctly

Given a slider-pk with a label prop
When the page loads
Then the input has an aria-label with that value

Given a slider-pk with disabled prop
When the page loads
Then the inner input has the disabled attribute

Given a slider-pk
When component initializes
Then data-ready="true" is set on the host
```

## Technical Notes
- Wraps native `<input type="range">` (accessible, keyboard navigable by default)
- Props: `min` (default 0), `max` (default 100), `step` (default 1), `value`, `name`, `disabled`, `label`, `variant` (default|primary)
- CSS: `@scope (.slider)` with vendor pseudo-elements (`::-webkit-slider-thumb`, `::-moz-range-thumb`)
- No JS needed for basic function — script only wires props and sets data-ready
