# S14-01 — Select/Listbox E2E + final migration polish

**Epic:** Listbox Migration
**Sprint:** Sprint 14
**Points:** 5
**Status:** done
**Priority:** Must

## User Story
As a developer using pseudo-html-kit, I want `select-pk` and `combobox-pk` to have verified E2E behaviour on Chromium, so that I can trust they work correctly with Popover API and native select enhancements.

## Context
Sprint 13 implemented the components. Sprint 14 adds E2E coverage and verifies the Popover API path for combobox and the `::picker(select)` path for select-pk. Combobox already uses `showPopover()`/`hidePopover()` programmatically (no `popovertarget` needed). Select uses native `<select>` with CSS `::picker(select)` via `@supports`.

## Acceptance Criteria

```gherkin
Given a combobox-pk on the page
When the user types in the input
Then the listbox opens and shows filtered options

Given a combobox-pk with options
When the user clicks an option
Then the input reflects the selection and listbox closes

Given a select-pk on the page
When the page loads
Then a native <select> is present with the correct options

Given a select-pk with required attribute
When the form is submitted empty
Then the native validation fires
```

## Technical Notes
- E2E tests go in `tests/form-components-e2e.e2e.js`
- Combobox opens via input focus/click → `showPopover()` call in script
- Select uses native `<select>` — test options rendering and value selection
- Add `data-testid` to the test page fixture for combobox and select

## Out of Scope
- Firefox/WebKit (Chromium-first policy)
- Full a11y audit (covered by unit tests)
